// GitHub'a Başlangıç Rehberi
Hiç GitHub kullanmadıysan buradan başla — sıfırdan ilk push'a kadar her şey burada.
G1
GitHub Hesabı Aç
github.com → Sign up
github.com adresine git ve "Sign up" butonuna tıkla.
E-posta adresi, şifre ve kullanıcı adı gir. Okul e-postanı kullanırsan GitHub Education avantajlarından yararlanabilirsin.
E-postana gelen doğrulama kodunu girerek kaydı tamamla.
💡 Kullanıcı adın profesyonel olsun — ileride CV'inde kullanacaksın.
G2
Git'i Bilgisayarına Kur
git-scm.com → Download
git-scm.com/downloads adresine git, işletim sistemine uygun versiyonu indir ve kur.
Kurulumdan sonra Terminal (Mac/Linux) veya Git Bash (Windows) uygulamasını aç.
Kurulumu doğrula:
$ git --version
# git version 2.x.x çıkması lazım
Git'e kimliğini tanıt (sadece bir kez yapılır):
$ git config --global user.name "Zeynep Çelik"
$ git config --global user.email "senin@email.com"
G3
GitHub'da Repository Oluştur
+ New repository
GitHub'a giriş yap → sağ üstteki "+" ikonuna tıkla → "New repository" seç.
Repository adı yaz (örn: azure-cicd-lab). Görünürlüğü Public olarak ayarla.
"Add a README file" kutusunu işaretle → "Create repository" butonuna bas.
✅ Repository hazır! Sayfadaki URL'yi kopyala: https://github.com/kullanici/repo.git
G4
Kodu GitHub'a Gönder
add → commit → push
Terminal'de proje klasörüne gir ve Git'i başlat:
$ cd proje-klasorum
$ git init
$ git remote add origin https://github.com/kullanici/azure-cicd-lab.git
Dosyaları ekle, kaydet ve gönder:
# Tüm dosyaları sahneye al
$ git add .

# Commit yap — değişikliği kaydet
$ git commit -m "ilk commit"

# GitHub'a gönder
$ git push -u origin main
🎉 Tebrikler! GitHub sayfanı yenile — kodların orada görünmeli.
📋
Git Komutları Kopya Kağıdı
En sık kullanacakların
git status
Hangi dosyalar değişti?
git add .
Tüm değişiklikleri hazırla
git commit -m "mesaj"
Değişiklikleri kaydet
git push
GitHub'a gönder
git pull
GitHub'dan güncelle
git log --oneline
Geçmiş commit'leri gör
// Adım Adım Kılavuz

1
Uygulama Hazırlığı Local
📁
Web Uygulaması Oluştur

Bilgisayarınızda basit bir web uygulaması oluşturun. Node.js, Python veya .NET kullanabilirsiniz.

✏️
Kimlik Bilgilerini Ekle

Uygulamanın ana sayfasına adınızı ve öğrenci numaranızı yazan bir metin ekleyin.

🔧
Git Deposu Başlat & GitHub'a Yükle

Projeniz için bir Git deposu başlatın ve kodlarınızı kişisel GitHub hesabınızdaki yeni bir repository'ye push edin.

# Yeni proje klasörü oluştur
$ mkdir my-web-app && cd my-web-app
$ git init
$ git remote add origin https://github.com/<kullanici>/<repo>.git
$ git push -u origin main
2
Azure App Service Kurulumu Cloud
🌐
Azure Portal'dan Web App Oluştur

portal.azure.com adresine girerek yeni bir App Service (Web App) kaynağı oluşturun.

⚡
Runtime Stack Seç

Yazdığınız dile uygun ortamı seçin: Node.js 20 LTS, Python 3.11 veya .NET 8.0

💰
Ücretsiz F1 Katmanı Seç

App Service Plan oluştururken F1 (Free) katmanını seçerek maliyeti sıfıra indirin.

F1 katmanı ücretsiz olup günde 60 dakika CPU süresi ve 1 GB depolama sunar — lab için yeterlidir.
3
Dağıtım Yapılandırması Deploy
🔌
Deployment Center'ı Aç

Azure Portal'da Web App'inizi açın → Sol menüden Deployment Center sekmesine gidin.

🔗
GitHub Kaynak Olarak Bağla

Kaynak (Source) olarak GitHub'ı seçin ve Azure'ın GitHub hesabınıza erişimine izin verin.

🌿
Repository & Branch Seç

GitHub repository'nizi ve main branch'inizi seçin. Azure otomatik olarak bir GitHub Actions .yml workflow dosyası oluşturacak.

# Azure tarafından otomatik oluşturulan workflow
on:
  push:
    branches:
      - main

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
4
CI/CD Sürecini Test Et Monitoring
🎨
Küçük Bir Değişiklik Yap & Push Et

Kodunuzda küçük bir değişiklik yapın — örneğin arka plan rengini değiştirin — ve git commit + push yapın.

👀
Actions Sekmesini İzle

GitHub repository'nizde Actions sekmesine gidin. Build ve Deploy adımlarının canlı olarak çalıştığını izleyin.

✅
Azure URL'den Doğrula

İşlem tamamlandığında Azure'daki URL'nizi tarayıcıda açın ve değişikliğin otomatik olarak yansıdığını doğrulayın.

Başarılı bir CI/CD pipeline: push → otomatik build → otomatik deploy → canlı URL güncellemesi
